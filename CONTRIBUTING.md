# Contribuer à Open Hémicycle

Merci de l'intérêt porté au projet. Open Hémicycle est un **bien public** : toute aide est bienvenue, en particulier la **vérification des données** et la **critique de la méthodologie**.

## Signaler une erreur dans les données

C'est la contribution la plus précieuse. Ouvrez une *issue* avec le label `data-correction` en indiquant :
- l'élément contesté (URL de la page, indicateur, date) ;
- la donnée correcte attendue ;
- une **source vérifiable**.

Toute correction validée est tracée dans le journal de corrections (transparence).

## Contribuer au code

1. Le code est sous **AGPL-3.0**.
2. Convention de commits : [Conventional Commits](https://www.conventionalcommits.org/) (`feat:`, `fix:`, `docs:`, `chore:`…).
3. Avant toute PR touchant un **indicateur**, lire [`docs/METHODOLOGY.md`](docs/METHODOLOGY.md) et [`docs/legal-guardrails.md`](docs/legal-guardrails.md).

## Règles éditoriales non négociables

- **Aucun adjectif moral** sur une personne (voir garde-fous juridiques).
- **Fait + source + date + lien** systématiques.
- **Symétrie** entre tous les groupes politiques.
- Un non-vote n'est **jamais** présenté comme une opposition.

## Note sur le développement autonome

Une partie du développement est assurée par un agent autonome (voir [`AGENTS.md`](AGENTS.md)) sous supervision humaine. Les décisions structurantes restent validées par un humain. Les contributions externes suivent le même cadre.
